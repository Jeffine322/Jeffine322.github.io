# 背景

如何让 Agent 从海量知识中，更准确地找到真正需要的内容，同时又不能让检索成本高得离谱



# RAG

RAG检索增强技术，原理：RAG 检索增强生成的基本流程并不复杂。首先将文本切割成若干个 chunk，再使用 Embedding 模型将用户的 query 和每个 chunk 分别转换为向量。通过计算 query 向量与 chunk 向量之间的余弦相似度，初步召回相似度较高的 Top-K 个 chunk。

但是，纯向量检索主要判断的是整体语义是否相近，很难准确识别否定、因果、条件和上下文逻辑。因此，还需要将 query 与召回的 chunk 共同输入 Rerank 模型，通过 Transformer 的 Attention 机制进行更细粒度的交互，最终输出两者的相关性分数，并根据分数重新排序。

传统RAG：

```
文档
 ↓
切分成很多 chunk
 ↓
为每个 chunk 计算 Embedding
 ↓
用户问题也计算 Embedding
 ↓
通过余弦相似度召回 Top-K
 ↓
使用 Rerank 重新排序
 ↓
交给大模型回答
```

这里涉及两种模型。

```
Embedding：先从海量文本中快速找出“看起来相关”的内容
    ↓
Rerank：再从候选内容中判断哪些“真正相关”
```

## 两种模型Embedding和Rerank

### Embedding

是输入一段文本（query 或 chunk），输出一个固定维度的向量

如：

```
“SQL注入如何判断列数”
        ↓
[0.12, -0.35, 0.68, ...]
```

优点：检索速度快、可以提前计算、几十万甚至几百万个chunk也能检索

缺点：**整段文本被压缩成了一个向量**，很多细节、否定关系、因果关系和逻辑关系可能会丢失

### Rerank

erank 不是分别计算 query 和文档的向量，而是把两者放在一起 (query, chunk)，然后Rerank 模型通常将 query 和 chunk 作为一组输入，通过 Transformer 对两段文本进行细粒度交互，并输出相关性分数。相比将两段文本分别压缩为独立向量，这种方式能够更充分地识别词语对应关系、否定表达、条件限制和上下文逻辑，因此通常具有更高的排序准确性。

 Rerank 通常比单纯的向量相似度更准确。但是贵

假设知识库有十万个 chunk，就不可能让用户每问一个问题，都把 query 和十万个 chunk 分别过一遍 Rerank。

所以传统方法一般只能：

```
Embedding 先召回 100 个
          ↓
Rerank 再从 100 个中选择 10 个
```

由此也出现了各种各样的优化方案，大部分都集中在 query 和 chunk 上，比如改写 query、优化 chunk 切分、增加关键词检索，或者使用 TextRank、PageRank 这类无监督算法对文本进行排序。

但是这些方法本质上还是在计算文本之间的相似度。只要 query 和 chunk 没有经过 Transformer 的 Attention 进行交互，就很难真正理解其中的否定、条件、因果和上下文逻辑，检索结果始终可能存在偏差。

那能不能绕过 Embedding，直接让 Rerank 模型对所有 chunk 进行判断呢？

显然不现实。一个知识库中可能存在几万甚至几十万个 chunk，如果每次查询都要将 query 和所有 chunk 分别输入 Rerank 模型，不仅计算成本非常高，检索时间也无法接受。

那能不能将知识组织成树状结构，让 Rerank 模型从根节点开始，一层一层地判断应该进入哪个分支，最终找到相关的 chunk？

这样确实可以减少每次需要参与 Rerank 的节点数量。但是新的问题也出现了：这棵树应该怎么构建？

如果建树时依然只是根据文本向量之间的相似度进行分类，那么本质上仍然是 Embedding 在决定知识之间的关系，只不过把原来的平面检索变成了树状检索，并没有真正解决语义逻辑上的问题。

所以现在真正需要解决的问题是：怎样构建一棵不仅依赖文本相似度，而且能够准确表示知识主题和知识关系的树？

# 建树思路

可以先对语义相近的 chunk 进行聚类，再利用 LLM 对同一聚类中的内容生成高度概括的摘要，并将摘要作为上层节点。通过递归执行聚类与摘要过程，可以逐步构建分层知识结构。

```
                    根节点
                 整个知识库摘要
                 /           \
          服务端安全          密码学
          /      \            /    \
      SQL注入   文件上传   代理签名  格密码
       /   \
   UNION注入  盲注
```

查询时，不再将用户问题与所有叶子 chunk 逐一比较，而是先对上层摘要节点进行 Rerank，再沿相关性较高的分支逐层向下检索，最终定位到具体的原始 chunk。

在这一过程中，Embedding 主要用于初步聚类，而检索阶段的路径选择则由 Rerank 模型完成。由于上层节点已经经过 LLM 的归纳和概括，其表达的不再只是局部文本内容，而是相应知识分支的整体主题，因此有望提高分层检索的准确性。

相比将所有知识组织在单棵树中，采用森林结构可以按照不同知识领域、数据来源或组织方式分别建树，从而降低单棵树规模过大带来的结构混乱。对于同时属于多个主题的知识，还可以通过节点复用、多路径索引或关联边进行补充。

该方案在理论上具有可行性，但初始构建过程需要多次执行聚类和摘要，耗时较长，计算成本也相对较高。不过，在新增知识时，通常只需要更新对应分支及其上层节点，无需重新构建整个知识库。

进一步研究后发现，这一思路与 RAPTOR（Recursive Abstractive Processing for Tree-Organized Retrieval）的核心方法较为接近。RAPTOR 通过递归聚类与摘要构建树状知识结构。在此基础上，还可以进一步研究多树组织、节点交叉关联、增量更新以及分层 Rerank 等优化方法。

# 继续优化

该方案虽然有望提高知识检索的准确性，但仍然存在两个主要问题：

1. 知识树的构建过程耗时较长，计算成本较高；
2. 父节点依赖 LLM 生成摘要，不仅会产生额外开销，还可能造成信息丢失。

这两个问题都与摘要过程密切相关。如果能够在降低摘要成本的同时，尽可能保留原始文本中的重要信息，就可以在一定程度上缓解知识树的构建压力。

因此，可以考虑对聚类后的多个 chunk 进行奇异值分解（SVD）。长文本中通常包含重复或贡献较低的内容，通过 SVD 提取主要语义成分，有可能过滤部分冗余信息，再将压缩后的文本交给 LLM 生成摘要。

相比直接使用 LLM 处理全部原始内容，SVD 的计算成本更低，也可能减少输入 Token 数量，从而降低摘要成本并减少无关内容的干扰。

基于这一思路，下面尝试实现一种基于 SVD 的文本压缩与摘要方法。

# 基于SVD奇异值分解的文本摘要算法

传统的无监督文本摘要算法的原理都很简单，对句子或词的频率，之间相似度来做取舍，来判断哪个句子重要然后保留，最终实现文本摘要。

这里可以用传统的TF-IDF值组成矩阵，例如列向量的句子，行向量是词，矩阵的元素值的含义就是，这个词在这个句子红的TF-IDF值(这个值的大致的作用是能区分这个词对全文的作用)。

然后奇异值分解，最终能得到一系列奇异值，根据奇异值能知道词的重要程度，但是现在要怎么拼回去得到一个摘要？如果要在摘要，那不就和原先的textrank和pagerank这种基于词频或者相似度打分的没有区别了吗？？因为理想效果是去掉废话，而不是将频率低的丢了，频率低的可能也很重要。



# embedding+SVD

传统无监督文本摘要通常根据词频、句子相似度或图排序结果，对句子的重要性进行评分，再从原文中选取部分高分句子组成摘要。

一种常见方法是使用 TF-IDF 构建词—句矩阵，其中行表示词，列表示句子，矩阵元素表示某个词在对应句子中的 TF-IDF 权重。随后对该矩阵进行奇异值分解，从中提取文本的主要潜在主题，并据此计算句子的重要程度。

但这种方法仍然存在一个问题：如何根据分解结果重新选出能够完整表达原文信息的句子？如果最终仍然按照词频、相似度或主题贡献对句子进行排序，那么它与 TextRank、PageRank 等传统抽取式摘要方法并没有本质区别。

我希望实现的并不是简单保留高频或高分句子，而是尽可能删除重复、空泛和信息贡献较低的内容，同时保留那些出现频率较低但具有重要事实、条件或结论的句子。因此，仅依赖 TF-IDF 和传统相似度评分可能无法达到预期效果。

# 总结

总体来看，该方法作为独立摘要算法时效果并不理想，主要问题是压缩比例有限，生成结果仍然较长。但它仍具有一定的应用价值，可以从以下两个方向使用。

1. 将其作为 LLM 摘要前的预处理步骤。先对较长的 chunk 进行信息筛选，删除部分重复和低贡献内容，再将压缩后的文本交给 LLM 生成摘要，从而减少输入 Token，降低计算成本，并减轻无关内容对摘要质量的干扰。
2. 将其应用于分层知识结构的构建。首先使用 Embedding 对 chunk 进行聚类，再对每个聚类中的文本进行压缩，形成一批上层节点。随后重复聚类与压缩过程，直到文本压缩幅度低于设定阈值，再引入 LLM 生成更准确的概括性摘要。之后还可以继续执行聚类与压缩，最终构建出分层知识森林。

因此，该方法更适合作为一种低成本的文本压缩工具，与 Embedding、LLM 摘要和分层检索结合使用，而不是直接替代完整的摘要模型。
